# PCCS 視覺化頁面 — 設計文件

日期：2026-06-02
狀態：待使用者最終審閱（v2 — 加入 Tone Map 區塊與已確認純色值）

## 概述與目標

在 jpca-practice 新增一個獨立頁面，視覺化 PCCS（日本色研配色體系）的「色相 × 色調」二維體系，作為 JPCA 考試的學習參考。顏色以螢幕可顯示的 sRGB 最接近值呈現，比對基準：

- `ref/IMG_PCCS_color.JPEG` — 教材色調圖
- `ref/tone-map.jpeg` — 日本色研官方「PCCS トーン別色相環」（Tone Map 區塊的排版依據）

純 vanilla HTML/CSS/JS、無建置步驟、無相依套件 — 符合本專案 GitHub Pages 直接部署 `src/` 的限制。

## 頁面結構

由上而下四個區塊：

| # | 區塊 | 內容 |
|---|------|------|
| 1 | 頁首 | 標題「PCCS 色彩體系」＋回首頁連結 |
| 2 | PCCS色調表（Tone Map） | 仿 `ref/tone-map.jpeg`：各色調花環＋v 大色環＋無彩色灰階軸 |
| 3 | 色彩互動區 | 左：色調圖（單色色塊）／右：24 色相環。點色相 → 色塊變色 |
| 4 | 色調印象卡片 | 12 張卡片，中文＋日文印象詞與色票 |

## 區塊 3：色彩互動區

### 色相環（右側）

- **24 個離散扇形色塊**，每塊 15°，單一純色（非漸層），SVG 繪製
- **8:Y（黃）固定在正上方**（12 點鐘位置）
- 色相編號**順時針遞增**：8:Y → 9:gY → ⋯ → 24:RP → 1:pR → ⋯ → 7:rY
- 每個扇形可點擊；目前選中的色相以外框＋粗體標籤標示

### 色調圖（左側）

- 12 個色調圓形色塊，依**彩度（橫軸）× 明度（縱軸）**二維配置：
  - 低彩度列（左）：p / ltg / g / dkg（上→下）
  - 中彩度列（中）：lt / sf / d / dk（上→下）
  - 高彩度列（右）：b / s / dp（上→下）、v（最右）
- 每個色塊顯示：色調記號＋中文名
- 座標軸標示：橫軸為分段標示，與色調欄位對齊 —「低彩度（2s）」對齊左欄、「中彩度（5s）」對齊中欄、「高彩度（8s・9s）」橫跨右側兩欄（仿 ref/IMG_PCCS_color 底部軸標）；縱軸「明度 高→低」

### 互動規格

| 操作 | 行為 |
|------|------|
| 頁面載入 | 預設選中 **8:Y**，色調圖顯示黃色系 12 色調 |
| 點擊色相環任一色相 | 色調圖 12 個位置即時更新為「該色相 × 各色調」的顏色 |
| 點擊色調圖任一色塊 | 彈出詳情面板：PCCS 記號（如 `v8`）、色調名（中日文）、色相名（日文）、HEX 值、**清濁分類** |
| 點擊詳情面板外部或關閉鈕 | 關閉詳情面板 |

無懸停（hover）依賴 — 所有互動皆以點擊觸發，行動裝置可完整操作。

## 區塊 2：PCCS色調表（Tone Map）

排版完全依照 `ref/tone-map.jpeg`（日本色研官方圖）：

### 構成元素

| 元素 | 規格 |
|------|------|
| **v 大色環** | 右側，**24 色相完整花環**（花瓣式色塊，已確認樣式），花瓣外側標色相編號 1–24，中心標「v ビビッド（さえた）」 |
| **11 個色調小花環** | **12 色相（偶數）花環**，花瓣外側標偶數編號，中心標色調記號＋日文名＋假名 |
| **無彩色灰階軸** | 最左側直向：W（ホワイト）→ ltGy（ライトグレイ）→ mGy（メディアムグレイ）→ dkGy（ダークグレイ）→ Bk（ブラック），含明度值標示（W9.5、Gy-8.5～Gy-2.5、Bk1.5）。9 個色卡均勻分布於圖表上下界之間 |

### 色調花環的空間配置（依 ref 圖，明度縱軸 × 彩度橫軸）

```
[灰階軸]   p     lt      b
  W                          \
  ltGy    ltg    sf    (s)    v（大色環）
  mGy                        /
  dkGy     g     d      dp
  Bk      dkg    dk
```

- **s（ストロング）**：ref 圖中僅為標籤圓圈（配色カード199a 不含 s 色票）。本頁面**以完整 12 色花環顯示**（用混色模型計算），但加註「※配色カード199a未収録」——比 ref 更完整，且保持教學一致性
- 花環之間以 ref 圖的相對位置擺放，視覺上呈現「越右越鮮豔、越上越明亮」

### Tone Map 互動

- 點擊**任何花瓣** → 詳情彈窗（同區塊 2：PCCS 記號、HEX、清濁分類、中日文名）
- 點擊灰階色塊 → 詳情顯示明度值（如 Gy-5.5）

> 變更紀錄（2026-06-02）：原設計的「點擊 v 大色環 → 跨色調同色相高亮」功能，經使用者要求移除。
> 變更紀錄（2026-06-02）：Tone Map 區塊移至頁面最上方、標題改為「PCCS色調表」、灰階色卡改為均勻分布。
> 變更紀錄（2026-06-03）：非 v 的 11 個小花環移除花瓣外側的色相編號（僅 v 大色環保留 1–24）。
> 變更紀錄（2026-06-03）：dkg 由濁色更正為**暗清色**（純色＋黑 80%）— dkg 位於色立體「黑 → 純色」下緣邊界，屬暗清色；濁色（中間色）為 s、sf、d、ltg、g 五個。

## 色彩資料規格（核心）

### 清濁色嚴格定義

| 分類 | 定義 | 對應色調 | 混色公式 |
|------|------|----------|----------|
| 純色 | 未混合 | v | 基底色（24 色，已確認） |
| 清色（明清色） | 純色＋白，無濁感 | b、lt、p | `C = V×(1−w) + 255×w`，b < lt < p（w 遞增） |
| 清色（暗清色） | 純色＋黑，無濁感 | dp、dk、dkg | `C = V×(1−k)`，dp < dk < dkg（k 遞增） |
| 濁色（中間色） | 純色＋灰 | s、sf、d、ltg、g | `C = V×(1−g) + G×g`，G 為各色調指定的灰階值 |

混色於 sRGB 通道層級進行（R、G、B 各通道分別套用公式）。

### 資料結構

```js
// src/js/pccs-data.js
const PCCS_HUES = [
  { num: 2, symbol: "2:R", jpName: "赤", vivid: "#e60033" },
  // ... 24 色相（奇數由內插函式生成）
];

const PCCS_TONES = [
  {
    id: "v", jpName: "ビビッド", jpDesc: "さえた", zhName: "鮮豔",
    category: "純色",            // 純色 | 明清色 | 暗清色 | 濁色
    mix: null,                    // { type: "white"|"black"|"gray", amount: 0–1, gray: "#..." }
    mapPosition: { col: 3, row: 2 },      // 區塊 2 色調圖位置
    toneMapPosition: { x: ..., y: ... },  // 區塊 3 Tone Map 花環位置
    impressions: { zh: [...], jp: [...] }
  },
  // ... 12 色調
];

const PCCS_GRAYS = [ { symbol: "W", value: 9.5, hex: "#f5f5f5" }, /* ... */ ];

// 任一色 = mixColor(getVivid(hueNum), PCCS_TONES[j].mix)
```

所有顏色由 12 基底色＋內插＋混色公式即時計算，不逐一硬編碼 — 保證清濁定義的嚴格一致性。

## 檔案結構

```
src/
├── pccs.html          # 新頁面（區塊 1–4 的 HTML 骨架）
├── css/
│   ├── style.css      # 既有檔案，不修改
│   └── pccs.css       # 新頁面樣式
├── js/
│   ├── main.js        # 既有檔案，不修改
│   ├── pccs-data.js   # 色彩資料：色相、色調、灰階、混色函式、印象詞
│   └── pccs.js        # 渲染：SVG 色相環、色調圖、Tone Map 花環、卡片、詳情彈窗、互動
└── index.html         # 修改：加入「PCCS 色彩體系」入口連結
```

模組邊界：

- `pccs-data.js` — 純資料與純函式（`mixColor`、`interpolateHue`、`getColor(toneId, hueNum)`），不碰 DOM。可獨立在 console 驗證。
- `pccs.js` — 只負責 DOM 渲染與事件，從 `pccs-data.js` 讀資料。
- 兩者以傳統 `<script>` 標籤依序載入（非 ES modules，避免 `file://` 開啟時的 CORS 問題）。

## 錯誤處理

- 純靜態頁面、無網路請求、無使用者輸入 — 無執行期錯誤情境
- 唯一防禦：`pccs.js` 啟動時檢查 `PCCS_HUES`／`PCCS_TONES` 已定義，否則顯示「資料載入失敗」訊息

## 測試方式

無測試框架（專案無 npm 相依）。驗證方式：

1. `npm run dev`（或 http://localhost:8080/pccs.html — 背景伺服器執行中）
2. 檢查清單：
   - [ ] 色相環為 24 等分離散色塊，8:Y 在正上方，順時針遞增
   - [ ] 預設載入時色調圖顯示 8:Y 的 12 色調
   - [ ] 點擊每個色相，色調圖即時更新
   - [ ] Tone Map：v 大色環（24 色）＋ 11 個小花環（12 色）＋ 灰階軸，位置依 ref 圖
   - [ ] 明清色（b/lt/p）與暗清色（dp/dk）無灰濁感；濁色（s/sf/d/ltg/g/dkg）有明顯灰味
   - [ ] 點擊任一色塊／花瓣顯示正確的 PCCS 記號、HEX、清濁分類
   - [ ] 12 張印象卡片中日文內容正確
   - [ ] 手機寬度（375px）下版面正常（Tone Map 可水平捲動）
   - [ ] 首頁連結可進入 PCCS 頁、PCCS 頁可返回首頁
3. 與 `ref/tone-map.jpeg`、`ref/IMG_PCCS_color.JPEG` 並列目視比對
