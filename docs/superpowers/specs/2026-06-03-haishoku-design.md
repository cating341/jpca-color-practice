日期：2026-06-03

## 概述與目標

在 jpca-practice 新增「基本配色法」介紹頁面（JPCA M1 初級範圍），內容依據 `ref/u5-color-schemes.json`（M1-U5 配色基礎單元，考試教材 source of truth）。每種配色法顯示定義、說明與 **2 個實際色卡範例**，色卡由既有 PCCS 色彩系統（`getColor`）即時渲染。

純 vanilla HTML/CSS/JS、無建置步驟 — 符合本專案 GitHub Pages 直接部署 `src/` 的限制。`ref/` 為 gitignore，JSON 內容轉寫為 JS 資料層（含補足範例），JSON 引用的示意圖（`./images/配色_*.jpg`）不存在也不使用。

## 領域規則（已與使用者確認）

- **類似色相配色 = 色相差 1~3**（依 JSON 教材定義；不細分隣接／類似，不含中差色相配色）
- 色相差為環狀計算（1–24 色相環，差值取 min(|a−b|, 24−|a−b|)）
- 無彩色記號：`N<明度值>`（如 N2、N6、N9.5）；`W` 視同 N9.5
- 色卡為純靜態展示（不可點擊）

```
│   └─ 色調配色     （同一／類似／對照 — 3 小節）
└─ 2. 基本配色技法  （超過兩個顏色的配色）
    ├─ 主調配色     （Dominant Color / Tone — 1 小節，3 色）
    ├─ 漸層配色     （色相／明度／彩度／色調 — 4 小節，4 色）
    ├─ 重點配色     （3 種情境，3 色）
    └─ 分離配色     （3 種情境，3 色）
```

頁面主標題為「配色基礎」。其下分兩大類別：**基本配色法**（兩個顏色的配色，以色相／色調的相對關係為基礎）與**基本配色技法**（超過兩個顏色的配色：主調、漸層、重點、分離等手法）。分類準則＝範例使用的顏色數量（2 色 vs 超過 2 色），與 JPCA「配色」與「配色技法」的理論分類一致；測試（`HAISHOKU_GROUPS`）強制保證此準則。

階層：大類別（`<h2 class="group-title">`，編號 1／2）→ 類別（`<h3 class="category-title">`）→ 小節（`<h4 class="scheme-title">`，中文＋英文）→ 定義說明 → 2 個色卡範例。

### 色卡版面（三種 layout）

| layout | 適用 | 呈現 |
|--------|------|------|
| `equal` | 色相／色調／主調／漸層配色 | 等寬色塊橫排；PCCS 記號不放在色塊上，於色塊下方註記列對齊標註 |
| `accent` | 重點配色 | 基調色（flex 5）＋配合色（flex 3）並排；**重點色作為小方形（46px、無邊框）疊在基調色塊內**，呈現「大面積中的小面積點綴」。資料順序為 [基調, 配合, 重點] |
| `separation` | 分離配色 | 主色（flex 4）＋分離色窄帶（flex 1）＋主色（flex 4） |

所有 layout 的色塊之間不留空隙（`gap: 0`），相鄰色塊直接相接。PCCS 記號改於色塊下方的註記列（`.scheme-note-row`）以等比例 flex 對齊標註，色塊本身不含任何文字。`accent` layout 的註記另標角色說明：基調欄（`.accent-note`，flex space-between）左標「基調 <記號>」、右標「重點 <記號>」——重點色註記靠右對齊疊於基調色塊右側的小方形，**不與基調合併**；配合欄為「配合 <記號>」（如 基調 N2 ／ 重點 v12 ／ 配合 N6）。角色詞（基調／配合／重點）以較淡的非等寬字（`.scheme-note-role`）呈現，與等寬的 PCCS 記號區隔。

## 內容資料（完整清單）

### 概說

> 配色是指兩個以上的色彩組合。好的配色能傳達特定的情感和印象。在此單元中，我們將學習基於 PCCS 色相與色調系統的基礎配色法則。

### 1. 色相配色

以色相環上的相對位置為基準的配色法。色相環上的距離決定配色性質。

| 小節 | 英文 | 定義 | 範例 1 | 範例 2 |
|------|------|------|--------|--------|
| 同一色相配色 | Identity Hue | 色相差 0（相同色相號碼）。統一、穩定但可能單調。 | v18＋p18（鮮豔藍＋淡藍） | dp8＋ltg8（深黃＋淺灰黃） |
| 類似色相配色 | Analogy Hue | 色相差 1~3。色相相近，易於調和且帶有些微變化。 | v10＋v12（黃綠＋綠） | sf12＋sf14（柔綠＋柔藍綠） |
| 對照色相配色 | Contrast Hue | 色相差 8~10。明顯對比感，視覺效果強烈。 | v6＋v16（黃橙＋綠藍） | dp8＋dp18（深黃＋深藍） |
| 補色色相配色 | Complementary Hue | 色相差 11~12（色相環正對面）。對比最強烈、刺激。 | v2＋v14（紅＋藍綠） | b8＋b20（明亮黃＋明亮藍紫） |

### 2. 色調配色

以 PCCS 色調（Tone）為基準的配色法。

| 小節 | 英文 | 定義 | 範例 1 | 範例 2 |
|------|------|------|--------|--------|
| 同一色調配色 | Identity Tone | 相同色調、不同色相。印象由該色調的特徵決定。 | v6＋v18（鮮豔橙＋鮮豔藍） | ltg8＋ltg20（淺灰黃＋淺灰紫） |
| 類似色調配色 | Analogy Tone | 色調圖上相鄰的色調。和諧且帶有層次。 | v22＋b22（鮮豔紫＋明亮紫） | sf8＋d8（柔黃＋濁黃） |
| 對照色調配色 | Contrast Tone | 色調圖上距離較遠（明度或彩度差異大）的色調。 | p20＋dkg20（淡藍紫＋暗灰藍紫） | v8＋ltg8（鮮黃＋淺灰黃） |

### 3. 主調配色

畫面中由某一特定色彩屬性佔據主導地位，產生強烈的整體統一感。M1 介紹基本概念，M2 會有更綜合的服裝應用。

| 小節 | 英文 | 定義 | 範例 1 | 範例 2 |
|------|------|------|--------|--------|
| 主調顏色／主調色調 | Dominant Color / Dominant Tone | 統一色相或統一色調，讓整體有一個明顯的「主角」。 | v18＋ltg18＋dp18（統一藍色相 Dominant Color） | v8＋v12＋v16（統一鮮豔色調 Dominant Tone） |

### 4. 漸層配色（Gradation / グラデーション）

顏色按照一定的規律作階梯式的變化。

| 小節 | 定義 | 範例 1（JSON 原有） | 範例 2（補足） |
|------|------|--------------------|----------------|
| 色相漸層 | 色相依序變化 | v6→v8→v10→v12（黃橙→黃→黃綠→綠） | v14→v16→v18→v20（藍綠→帶綠藍→藍→藍紫） |
| 明度漸層 | 明度由亮到暗或由暗到亮變化 | p8→lt8→b8→dp8（由淺黃到深黃） | lt16→sf16→d16→dk16（由淺藍到暗藍，明度遞減） |
| 彩度漸層 | 彩度由高到低或由低到高變化；註明彩度高低不等於清濁（清濁由加白／黑 vs 加灰決定，非彩度），如 p 與 ltg 彩度相近但清濁相反 | v24→b24→sf24→ltg24（鮮豔紅紫→明亮→柔→淺灰） | v18→s18→d18→g18（鮮藍→強藍→鈍藍→灰藍） |
| 色調漸層 | 色調圖上有規律的移動 | p16→lt16→b16→v16（淡→淺→明亮→鮮豔） | v4→dp4→dk4→dkg4（鮮豔→深→暗→暗灰） |

### 5. 重點配色（Accent / アクセントカラー）

在單調或大面積的統一配色中，加入小面積的強烈對比色，達到畫龍點睛的效果。**面積最大的稱為基調色（Base color）**、**次要的稱為配合色（Assorted color）**、**面積最小的為重點色（Accent color）**。

| 情境 | 範例 1（JSON 原有） | 範例 2（補足） |
|------|--------------------|----------------|
| 無彩色＋高彩度重點色 | N2／N6／v12（灰底＋鮮綠重點） | N9.5／N5.5／v18（白灰底＋鮮藍重點） |
| 低明度配色＋高彩度重點色 | dkg16／dk16／v8（暗藍底＋鮮黃重點） | dkg20／dk20／v6（暗藍紫底＋鮮橙黃重點） |
| 低彩度明度差小的配色＋高彩度暖色系重點色 | ltg12／ltg14／v6（灰綠底＋鮮橙黃重點） | ltg16／ltg18／v4（灰藍底＋鮮橙重點） |

範例順序固定為：基調色、配合色、重點色。

### 6. 分離配色（Separation / セパレーション）

在兩種對比過強或過於模糊的顏色之間，加入無彩色或低彩度顏色作分隔，使整體和諧或輪廓清晰。

| 情境 | 範例 1（JSON 原有） | 範例 2（補足） |
|------|--------------------|----------------|
| 高彩度強烈配色＋無彩色分離色 | v18／N9.5／v6（藍／白分離／橙） | v8／N1.5／v20（黃／黑分離／藍紫） |
| 低彩度模糊配色＋低明度分離色 | p8／N2／p10（淡黃／深灰分離／淡黃綠） | p16／N2／p18（淡藍／深灰分離／淡藍紫） |
| 低明度配色＋高明度分離色 | dp16／N9.5／dp20（深藍／白分離／深藍紫） | dk10／N9.5／dk14（暗黃綠／白分離／暗藍綠） |

範例順序固定為：主色、分離色、主色。
（JSON 原文的 `W` 記號統一轉寫為 `N9.5`。）

## 資料層 API（`src/js/haishoku-data.js`）

依賴 `pccs-data.js`（瀏覽器 `<script>` 順序載入；Node `require`），沿用 `quiz-seidaku-data.js` 的 `_pccs` shim pattern。

```js
// 記號解析
parseColorNotation("v2")     // → { type: "chromatic", toneId: "v", hueNum: 2 }
parseColorNotation("ltg20")  // → { type: "chromatic", toneId: "ltg", hueNum: 20 }
parseColorNotation("N2")     // → { type: "neutral", value: 2 }
parseColorNotation("W")      // → { type: "neutral", value: 9.5 }
parseColorNotation("xx")     // → throw（描述性錯誤）

// 顏色計算
getSchemeColor("v2")         // → hex（chromatic：呼叫 getColor；neutral：呼叫 getNeutralColor）
getNeutralColor(2)           // → hex（PCCS_GRAYS 線性插值；超出範圍 clamp 到 [1.5, 9.5]）

// 資料
HAISHOKU_OVERVIEW            // 概說文字
HAISHOKU_GROUPS              // 2 大類別：[{ id: "basic"|"technique", title, description, categoryIds: [...] }]
HAISHOKU_CATEGORIES          // 6 類別：[{ id, title, titleEn, description, layout, schemes: [...] }]
//   每個 scheme：{ title, titleEn, description, examples: [{ colors: ["v2","p2"], label }, ...] }
```

### 無彩色灰階插值

`PCCS_GRAYS` 只有 9.5／8.5／…／1.5 等 0.5 結尾的值。`getNeutralColor(value)`：

1. 找出 `PCCS_GRAYS` 中夾住 value 的上下兩筆，依比例線性插值 RGB
2. value 恰好命中既有值 → 直接回傳該 hex
3. value 超出 [1.5, 9.5] → clamp（N0 視為 1.5、N10 視為 9.5）

例：N2 介於 2.5（#404040）與 1.5（#262626）正中間 → `#333333`。

## 渲染層（`src/js/haishoku.js`）

- 防禦：檢查 `HAISHOKU_CATEGORIES`／`getSchemeColor` 已定義，否則顯示「資料載入失敗」（沿用既有 pattern）
- 依 `HAISHOKU_GROUPS` 渲染兩個大類別（編號 1／2），各大類別依 `categoryIds` 取出並渲染其下 `HAISHOKU_CATEGORIES` 類別與小節
- 每個範例依其 category 的 layout（equal／accent／separation）渲染色卡
- PCCS 記號渲染於色塊下方的註記列（`.scheme-note-row`），與色塊以等比例 flex 對齊；色塊本身不含文字
