# 清濁度測驗單元 — 設計文件

日期：2026-06-03
狀態：已經使用者口頭核准設計，待審閱本文件

## 概述與目標

在 jpca-practice 新增一個獨立的測驗頁面：**清濁度測驗**。每題顯示四個「相同色相、不同色調」的色塊，其中一個的清濁屬性（清色／濁色）與其他三個不同，使用者選出後送出公布答案，可連續作答。

目的：訓練「目視分辨清色與濁色」的能力 — 個人色彩診斷（黃底／藍底、四季分類）的核心技能。

純 vanilla HTML/CSS/JS、無建置步驟、無相依套件 — 符合本專案 GitHub Pages 直接部署 `src/` 的限制。

## 清濁分類（領域規則）

由 `PCCS_TONES` 既有的 `category` 欄位推導：

| 清濁 | 分類 | 色調 | 數量 |
|------|------|------|------|
| **清色** | 明清色（純色＋白） | b、lt、p | 6 |
| | 暗清色（純色＋黑） | dp、dk、dkg | |
| **濁色** | 中間色（純色＋灰） | sf、d、ltg、g | 4 |
| （排除） | 純色 | v | — |
| （排除） | 濁色但不出題 | s | — |

> 變更紀錄（2026-06-03）：dkg 由濁色更正為暗清色（使用者指正）— dkg 位於色立體「黑 → 純色」下緣邊界。
> 變更紀錄（2026-06-03）：s（ストロング）排除於題目之外（使用者要求）— s 加灰量僅 20%，視覺上接近清色，且為配色カード199a未収録色調。s 在 PCCS 分類上仍屬濁色（PCCS 頁面不受影響）。

**v（ビビッド）排除**：依嚴格 PCCS 定義，純色既不屬清色也不屬濁色，不出現在題目中（使用者已確認）。
**s（ストロング）排除**：加灰量僅 20%，視覺上接近清色，不適合作為清濁判斷題目（使用者已確認）。
題目只從其餘 10 個色調選取。

## 出題邏輯（鬆散相鄰加權）

使用者已確認採「鬆散相鄰」：不要求嚴格連通，但加權偏好色調表上距離近的組合。

1. **題庫枚舉**（載入時計算一次）：
   - 所有「1 清 + 3 濁」組合：C(6,1) × C(4,3) = 24 組
   - 所有「1 濁 + 3 清」組合：C(4,1) × C(6,3) = 80 組
   - 共 104 組，每組記錄答案（清濁屬性不同的那一個）
2. **緊湊度加權**：
   - 每組以色調表格座標（欄、列）計算分散度 = 四色調兩兩距離總和
   - 抽樣權重 = `1 / spread²` — 相鄰緊湊的組合最常出現，分散的組合偶爾出現
3. **每題產生**：
   - 隨機（加權）抽一組色調組合
   - 隨機色相（1–24）
   - 隨機打亂四個色塊的顯示順序
   - 顏色以既有 `getColor(toneId, hueNum)` 計算

### 色調表格座標（緊湊度計算用）

依既有色調圖（spotPos）的欄列關係，排除 v：

```
        欄0（低彩度）  欄1（中彩度）  欄2（高彩度）
列0     p            lt           b
列1     ltg          sf           s
列2     g            d            dp
列3     dkg          dk           —
```

距離 = 歐氏距離（欄差、列差）。高彩度欄（b/s/dp）實際縱向位置有半格偏移，計算時忽略此細節（鬆散相鄰不需精確）。

```
src/
├── quiz-seidaku.html            # 測驗頁面
├── css/
│   └── quiz-seidaku.css         # 頁面樣式
├── js/
│   ├── quiz-seidaku-data.js     # 純邏輯：清濁分類、題庫、出題、判分 — 不碰 DOM
│   └── quiz-seidaku.js          # DOM 渲染與互動
├── index.html                   # 修改：topic-nav 加入「清濁度測驗」卡片
tests/
└── quiz-seidaku-data.test.js    # 資料層測試（node assert，無框架）
```

模組邊界（沿用 PCCS 頁面 pattern）：

- `quiz-seidaku-data.js` — 純資料與純函式，依賴 `pccs-data.js`（瀏覽器以 `<script>` 順序載入；Node 以 `require` 載入測試）。隨機數以參數注入（`rng` 函式，預設 `Math.random`），使測試可重現。
- `quiz-seidaku.js` — 只負責 DOM 渲染與事件。
- 載入順序：`pccs-data.js` → `quiz-seidaku-data.js` → `quiz-seidaku.js`。

### 資料層 API

```js
// 清濁分類
function isClearTone(toneId)      // b/lt/p/dp/dk/dkg → true；s/sf/d/ltg/g → false；v → throw
function seidakuLabel(toneId)     // "清色（明清色）" | "清色（暗清色）" | "濁色（中間色）"

// 題庫（載入時計算）
var SEIDAKU_COMBOS = [ { tones: ["lt","sf","d","g"], oddTone: "lt", weight: ... }, ... ]

// 出題
function generateQuestion(rng)    // → { hueNum, choices: ["sf","lt","d","g"], answerIndex: 1 }
```

## UI 與互動流程

```
┌──────────────────────────────────┐
│ 清濁度測驗            ← 回到首頁   │
├──────────────────────────────────┤
│  下面四個顏色中，哪一個的清濁屬性    │
│  和其他三個不同？                  │
│                                  │
│         連續答對：3               │
│                                  │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐    │
│  │    │ │    │ │    │ │    │    │  ← 純色塊，無文字標示
│  └────┘ └────┘ └────┘ └────┘    │
│                                  │
│         [ 送出答案 ]              │
└──────────────────────────────────┘
```

| 操作 | 行為 |
|------|------|
| 頁面載入 | 自動出第一題 |
| 點擊色塊 | 選取（高亮邊框），可改選 |
| 未選擇時 | 「送出答案」按鈕禁用 |
| 送出答案 | 公布結果與解說；色塊不可再點選 |
| 答對 | 連續答對數 +1 |
| 答錯 | 連續答對數歸零 |
| 公布後 | 按鈕變為「下一題」，點擊重新出題 |

無懸停（hover）依賴 — 所有互動皆以點擊觸發，行動裝置可完整操作。

## 答案公布（解說）

公布後每個色塊下方顯示：

- 色調記號＋日文名＋中文名（如 `lt ライト 淺`）
- 清濁分類標籤：明清色／暗清色／濁色
- 正確答案標 ✓；使用者答錯時，其選擇標 ✗

色塊下方另顯示一行總結解說，例如：

> **sf（柔和）是濁色**（純色＋灰），其他三個都是清色（純色＋白或＋黑）。

## 響應式設計

- 桌面：四個色塊橫排一列
- 窄螢幕（< 600px）：色塊 2×2 網格排列
- 色塊尺寸足夠大（≥ 120px），確保顏色判斷不受面積影響

## 錯誤處理

- `quiz-seidaku.js` 啟動時檢查 `PCCS_TONES`／`generateQuestion` 已定義，否則顯示「資料載入失敗」訊息（同 PCCS 頁面 pattern）
- `isClearTone("v")` 拋出錯誤（v 不應出現在測驗中，出現即為程式錯誤）

## 測試方式

### 資料層自動測試（`node tests/quiz-seidaku-data.test.js`）

- [ ] 清濁分類：分類正確（清色 6：b/lt/p/dp/dk/dkg；濁色 5：s/sf/d/ltg/g）、v 拋出錯誤
- [ ] 測驗色調 10 個（排除 v 與 s）
- [ ] 題庫共 104 組，每組剛好 4 個色調、3:1 清濁比例、`oddTone` 正確
- [ ] 題庫不含 v 與 s
- [ ] `generateQuestion`（注入固定 rng）：四色調互異、`answerIndex` 指向清濁不同的那一個
- [ ] 加權抽樣：以大量抽樣統計，緊湊組合（如 lt/sf/d/g）出現頻率高於分散組合（如 p/s/d/g）

## 不在範圍內（YAGNI）

- 回合制計分（10 題統計）
- 難度切換（顯示／隱藏色調記號）
- 作答紀錄持久化（localStorage）
- 其他屬性測驗（明度、彩度、色相）— 未來可仿此 pattern 擴充
